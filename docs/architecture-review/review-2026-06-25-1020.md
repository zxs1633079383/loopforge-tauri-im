## 2. 具体边界发现（file:line + 建议）

### V1 · 转发路径在前端组装 wire body（最明确的一条）— MINOR
- **位置**：`src/app/app.component.ts:790-792`（`onForward()`），配合 `:786-789` 的协议决策注释。
- **问题**：前端直接构造出站 `posts[]` 载荷 `[{ message, temporaryId, type }]`，并内嵌服务端协议知识——`:786-789` 长注释据 `entity/post.go:188` 的 PreSave id 语义，自行决定「转发是建新消息、posts 元素只携内容不携源 id」。这与全仓其余命令的铁律「壳不臆造 body（body 由 Rust/helix 拼）」相悖：唯独转发把 wire Post 对象的拼装放在了 Angular 里。
- **应在何处**：`src-tauri/commands.rs` 的 `im_relay_messages` + helix-im `outbound/posts_relay.rs`。前端应只传 `{ 源消息引用（postId/text/type）, channelIds }`，由 Rust 组装 `posts[]`（含 temporaryId 生成与「不带源 id」决策），与其余命令对齐。
- **建议**：把 Post 对象组装与「省略源 id」协议决策移入 Rust 命令；前端 `onForward` 退回为纯参数转发。

### V2 · 客户端协议 ID 生成，且重复 — MINOR（可辩护模式 + DRY 问题）
- **位置**：`src/app/im/im-store.service.ts:1804-1813`（`genTempId()`）与 `src/app/app.component.ts:797-802`（`genForwardTmp()`）。
- **问题**：两处都硬编码 z-base-32 字符集 + 26 位格式（= 现网 server id 格式 / 乐观锚协议），属协议知识落在视图层；且两份实现重复（违 DRY）。`temporary_id` 是发送协议的 upsert 锚，其格式约定本质是协议契约。
- **应在何处**：helix 侧已有 `IdSource`/`Random` port（`crates/helix-driver-instrument/src/id.rs`）——temporary_id 理应由 Rust 经 IdSource 生成并随 `im_send`/relay 回传（同时利于 tape 录放确定性，呼应 CHECKLIST「IdSource 注入缝」待续项）。
- **建议**：中期把 tmp_id 生成下沉 Rust（IdSource）；短期至少把两处合并为单一共享 util，消除重复。注：乐观 UI 客户端生成临时 id 是常见可辩护做法，故列为 advisory 而非硬违反——核心顾虑是「格式协议知识 + 重复」。

### V3 · 成员集重建复刻后端域逻辑 — MINOR
- **位置**：`src/app/im/im-store.service.ts:1368-1428`（`applyMemberUpdated()` + `extractMemberIds()` + `memberChangeField()`）。
- **问题**：从后端全量 channel 帧在 JS 里重建频道成员名册——`join[] + members[]/adminUsers[]/boss[]/owner − leave[]`，注释明确承认「与 helix `channel_write::collect_members` 同口径」。这是「谁属于该频道」的**域计算**落在视图层（即便是渲染路径，也复刻了后端成员收敛语义）。
- **应在何处**：helix-im 投影层。权威成员 id 集应由 Rust 投影成扁平 member-id 列表下发，前端只 map 成行渲染，而非从 raw channel 帧自行收敛。
- **建议**：让 `emit_channel_member_updated` 投影吐出已收敛的成员 id 集；前端 `applyMemberUpdated` 退化为纯透传渲染。

### V4 · 分页 pivot 锚选取在前端 — MINOR（边界性）
- **位置**：`src/app/im/im-store.service.ts:709-730`（`loadOlder()`）。
- **问题**：遍历当前行集挑「最旧（createAt 最小且有 server id）」作为上拉翻页锚。翻页编排（轮数/before/凑够判定）已正确放在 helix，但锚选取这一步是落在前端的小块分页决策。
- **应在何处**：可接受现状（薄壳只选锚 + 入泵）；若逻辑增长则应下沉 Rust。
- **建议**：暂列 advisory，保持监控；不必立即改。

### 已知且已论证的例外（非待修违反，仅记录）
- `src/app/im/im-store.service.ts:491-502`（`setManger()`）这段审计记录描述的是一条**现已废弃的 `data-admin` 乐观本地刷**历史路径：当时因 add/remove manger 的后端 WS 缺口、L1 单账号下 ② 投影不到达，文档保留了该偏离作为快照；当前口径已改为只认 L2 #45 广播帧对账的权威态。

### 正面确认

- `tauri-bridge.service.ts` 纯 transport（含非 Tauri 降级），`props-extract.ts` / `read-result-extract.ts` 纯函数抽取，`message-row.model.ts` / `projection.types.ts` 纯类型，均无业务逻辑。
