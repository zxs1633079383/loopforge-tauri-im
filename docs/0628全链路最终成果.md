## 3. 关键架构发现(高价值)

- **send round-trip 真根因 = helix 客户端 echo 对账**,不是后端零广播、不是 HoL。裁决 agent 深 trace 锁定:服务端 produce→consume→broadcast→socket-write 全绿、client 真收到 934 字节 post 帧,但 `reconcile_send_http` 提前 `pending_sends.remove` + cursor-gate buffer 双堵 → server_id 永不落乐观行。早先绿是因当时无 cursor gap(两路有一条通),seeded/trim 后有 gap → 两路全堵。
- **WS echo 不丢铁律**(用户拍板 · 百万长连接):入队即不丢 / 背压只 DisconnectSlow kick 不静默丢 / 实时 echo 优先于冷启 sync(de3d359 QoS 双车道 · 防 head-of-line blocking)。hub 层本已守住"入队不丢",缺的是单条有序 wire 的排队公平性。
- **Pulsar 共享裸 topic 偷台隐患**:`post_sender-1` 裸 topic + KeyShared 共享订阅 → 别的 pod/旧 consumer 偷消息致间歇零广播。dev 用 USER 后缀 + Exclusive 订阅隔离(`im-pulsar-local-isolate`)。
- **4 跳诊断框架**(别 thrash):① 后端出站推送 → ② helix 收到 → ③ helix 输出投影(或被 cursor-gate buffer 不吐)→ ④ UI 渲染。每跳加常驻日志,任一失败一眼定位卡哪侧,不再反复猜。隐藏跳:②→③ 的 cursor-gate、① 里的连接路由。
- **C004 实操**:#14 已读 ④ 发现冻结 oracle 自相矛盾(`batch_upsert` 但 read 是幂等全量覆盖 = `batch_update`),agent 停下标人审、不擅改;用户批准后才改。错 oracle 也不自产自判。

### 6.2 echo gate-gap 整类根治(4 层逐跳)
1. **helix `reconcile_send_http`**:SUCCESS-无serverId 分支不提前清 pending_sends → 留给 WS echo 幂等补 server_id。
2. **helix `gate.rs` post_update(type2)**:cursor-gap 下兜底 emit `im:post:updated`(对齐 `post.rs` type1·不推 cursor)。
3. **helix `trigger_backfill_if_large_gap`**:内容帧 gap>2 当步立即触发该频道 proactive resync 追平 cursor(守 HX-C008·真填缺口不绕 gate)+ 防风暴三重守卫(inflight_sync / SyncScheduler K=8 / 小缺口阈值)。
4. **cses 真根(成员口径分裂)**:WS 推送 `GetChannelIdsInUsers`(**不过滤 delete_at**)vs sync 越权闸门 `FilterMemberChannelIds`/`GetRootMemberChannelIds`(**过滤 delete_at=0**)→ **软删成员 WS 收 echo 但 sync 返 no_change/nextSeq=fromSeq → cursor 永卡 → buffered echo 永不 flush**。user 444 实测 **510 行软删成员 / 21 行有效**。修=去 `delete_at=0` 谓词与 WS 推送同源(「WS 推得到的频道·sync 必放行其缺口」by construction · 真非成员仍拦 · WS 推送零改 · hot-path O(1) 不破)。

### 6.3 诊断方法论沉淀(用户框架 · 别 thrash)
**4 跳逐跳常驻日志**:① 后端出站(cses broadcast) → ② helix 收到(`HOP2 {action,seq}`)→ ③ helix 输出/gate(`HOP3 apply-emit | buffer-gap`,区分"没收到"vs"收到但 gate buffer 不吐")→ ④ UI 渲染(run.jsonl)。一眼定位卡哪侧。
- **关键基建发现**:WS 入站走 native `Tick::Inbound` **绕过 `Recording::recv()` tee** → run.jsonl 曾永无 ws-recv 帧(②面盲点·这正是之前"笼统说后端缺口"查不下去的根因)。已补 `HOP2`/`HOP3` 常驻 tracing。
